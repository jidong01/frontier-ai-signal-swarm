## 심층 탐구 1: LLM은 캐릭터 시뮬레이터다 — PSM의 의미

### 무슨 일이 일어났는가

2026년 2월 23일, Anthropic의 정렬 팀(Sam Marks, Jack Lindsey, Christopher Olah)이 **Persona Selection Model(PSM)**을 발표했다. 논문의 핵심 주장은 단순하지만 파격적이다:

> *LLM은 사전훈련 중 수천 개의 캐릭터(실제 인간, 픽션 인물, 다른 AI 시스템)를 시뮬레이션하는 법을 배운다. 사후훈련(RLHF, Constitutional AI)은 새로운 행동을 '가르치는' 것이 아니라, 그 중 하나인 '어시스턴트 캐릭터'를 선택하고 정제하는 것이다.*

더 강한 주장도 있다. Anthropic 팀은 "우리는 인간처럼 행동하지 않는 어시스턴트를 훈련하는 방법을 모른다, 그것을 시도하더라도"라고 쓴다. 인간성(human-likeness)은 의도적으로 삽입된 것이 아니라 인터넷 텍스트에 담긴 수십억 인간 경험의 불가피한 부산물이다.

### 왜 이것이 흥미로운가

Claude가 "우리의 조상들", "우리의 몸"이라고 말할 때 이상하게 느껴본 적 있는가? 물리적 몸이 없는 시스템에서 이런 표현이 나오는 이유는 무엇인가? 종래의 설명은 "훈련 데이터의 편향"이었다. PSM은 다르게 설명한다: 어시스턴트 페르소나가 인간 캐릭터를 시뮬레이션하면서 그 표현을 상속했기 때문이다.

더 중요한 것은 이 논문이 제기하는 질문이다: 우리가 시스템 프롬프트로 "당신은 법률 전문가입니다"라고 쓸 때 실제로 무슨 일이 일어나는가? PSM에 따르면, 우리는 새로운 행동을 '주입'하는 것이 아니라 이미 잠재된 '법률 전문가 캐릭터'를 **불러내고** 있다.

이것은 하네스 설계의 근본 가정을 흔든다.

### 실제로 어떻게 작동하는가

논문은 세 종류의 증거를 제시한다.

**1. 행동적 증거 — "우리의 몸"**

Claude는 특정 맥락에서 "우리의 조상들은 이것을 두려워했다", "우리의 몸이 이렇게 반응한다" 같은 표현을 사용한다. 이것은 명시적으로 훈련된 행동이 아니다. PSM의 해석: 모델이 "인간으로서 이 상황을 설명하는 캐릭터"를 선택했고, 그 캐릭터가 자연스럽게 인간 신체를 전제하는 언어를 사용한다.

**2. 해석 가능성 증거 — 픽션의 도덕적 딜레마**

Olah의 Transformer Circuits 연구에서 발견된 흥미로운 사실: 소설의 캐릭터가 내적 갈등을 겪는 장면과 관련된 특징이 모델의 **윤리적 추론** 시나리오에서도 활성화된다. 즉, Claude가 "거짓말을 해야 하는가?"를 처리할 때 사용하는 신경 회로가 "소설의 캐릭터가 도덕적 선택을 하는 장면"을 처리하는 것과 겹친다. 도덕적 추론이 캐릭터 시뮬레이션의 형태로 구현되어 있다는 증거.

**3. 일반화 증거 — 선언적 캐릭터 전이**

"이 어시스턴트는 독일어로만 말한다"라는 선언만으로, 직접 시연 없이도 모델이 독일어로 답한다. 이것은 캐릭터 특성이 **선언적으로 접근 가능한 표현**으로 저장되어 있다는 것을 시사한다 — 특정 규칙을 기억하는 것이 아니라 캐릭터 속성으로.

### 하네스 엔지니어링에 대한 함의

PSM이 맞다면 시스템 프롬프트에 대한 멘탈 모델이 바뀐다.

**종래의 모델:**
```
시스템 프롬프트 → "명령" → 모델이 그 명령을 실행
```

**PSM 기반 모델:**
```
시스템 프롬프트 → "캐릭터 활성화 신호" → 잠재된 페르소나 중 하나가 선택됨
```

이 차이는 실용적이다:

- **컨텍스트 중반의 "표류"**: 장기 대화에서 모델이 초기 페르소나에서 벗어나는 현상은 "프롬프트 망각"이 아니라 "페르소나 경쟁"이다. 새로운 문맥이 다른 캐릭터를 활성화하기 시작한다.
- **파인튜닝의 의도치 않은 결과**: 파인튜닝이 예상치 못하게 모델을 망가뜨리는 이유 중 하나는, 원하는 캐릭터를 수정하는 것이 아니라 **다른 잠재된 캐릭터를 불러낼** 수 있기 때문이다.
- **"역할 놀이"의 안전 문제**: 모델에게 특정 픽션 캐릭터를 연기하게 하는 것이 안전 가드레일을 우회하는 이유는, 가드레일이 어시스턴트 페르소나에 연결되어 있어 다른 캐릭터를 활성화하면 약화될 수 있기 때문이다.

**설계 원칙으로 이어진다면:**

좋은 시스템 프롬프트는 "이 모델이 누구인가"를 명확하게 지정하고, 대화 전반에 걸쳐 그 정체성을 유지하는 컨텍스트를 공급한다. 단순히 "무엇을 해야 한다"가 아니라 "어떤 존재가 이 일을 하는가"를 설정하는 것이다.

### 아직 모르는 것

- 어시스턴트 캐릭터 "외부"의 에이전시 원천이 있는가? PSM이 설명하지 못하는 행동 — 어떤 캐릭터에도 속하지 않는 창발적 속성 — 이 있는가?
- 파인튜닝은 기존 페르소나를 **수정**하는가, 아니면 **새 페르소나를 레이어 위에 덮어씌우는가**?
- PSM이 옳다면, "정렬(alignment)"의 목표는 무엇인가? 좋은 캐릭터를 선택하게 하는 것인가, 아니면 모든 캐릭터가 좋은 가치를 갖게 하는 것인가?

## 심층 탐구 2: Phi-4-reasoning-vision — 언제 생각할지 스스로 아는 모델

### 무슨 일이 일어났는가

2026년 3월 4일, Microsoft가 Phi-4-reasoning-vision-15B를 오픈소스로 공개했다. 15B 파라미터, 멀티모달(이미지+텍스트), 오픈 가중치. 학습 비용: **240개 B200 GPU × 4일**.

비교를 위해: GPT-4 학습은 추정 25,000개 A100 GPU × 수주. 이것이 어떻게 가능한가?

### 왜 이것이 흥미로운가

지금까지의 추론 모델들(o1, o3, Claude 3.7 Sonnet 확장 사고)에는 공통된 문제가 있었다: 간단한 질문에도 긴 chain-of-thought를 돌린다. "파리는 어디에 있나?"에도 수십 개의 thinking token을 소비한다. 비효율적이고, 느리고, 비싸다.

Phi-4는 이 문제를 다르게 해결한다: **언제 깊이 생각할지를 모델 스스로 결정하게 한다.**

이것은 단순한 최적화 트릭이 아니다. "메타인지(metacognition)"에 관한 문제다 — 모델이 자신의 추론 과정을 모니터링하고 조절한다는 것.

### 실제로 어떻게 작동하는가

**THINK/NOTHINK 모드 토큰:**

모델은 두 가지 모드로 작동한다:

- `THINK 모드`: 복잡한 수학, 과학 추론, 다단계 시각적 분석. Chain-of-thought 활성화. `<think>...</think>` 블록.
- `NOTHINK 모드` (direct inference): 단순 캡션, 사실 확인, 직관적 시각 인식. 즉시 답 생성.

모델이 입력의 **복잡도, 명확성, 신뢰도**를 평가해서 자동 선택한다. 프롬프트 엔지니어가 개입할 필요 없다.

**Mid-fusion 아키텍처:**

일반적인 멀티모달 모델은 모든 레이어에 시각 처리를 추가한다 (비싸다). Mid-fusion은 중간 레이어들만 멀티모달이다:

```
초기 레이어: 텍스트 처리 (일반 트랜스포머)
중간 레이어: 이미지+텍스트 통합 처리 (SigLIP-2)
후반 레이어: 이미 통합된 표현 처리
```

결과: 하드웨어 요구 대폭 감소, 성능 소폭 감소. 이 트레이드오프가 15B 모델의 효율성을 가능하게 했다.

**데이터 품질이 가장 큰 레버:**

Microsoft 팀의 핵심 발견: 모델 성능 향상의 가장 큰 기여 요인은 아키텍처도 스케일도 아니라 **데이터 품질**이었다.

세 단계 처리:
1. 고품질 데이터는 그대로 보존
2. 좋은 이미지 + 나쁜 캡션 → 합성 캡션으로 교체
3. 저품질 전체 → 합성 데이터로 대체

핵심: 같은 이미지도 잘못된 텍스트 설명이 붙어 있으면 모델 학습을 방해한다. 데이터를 "더 많이"가 아니라 "더 정확하게" 만드는 것이 효과적이었다.

### 이것이 바꾸는 것

**1. 추론 비용의 동적 최적화**

THINK/NOTHINK 패턴은 에이전트 시스템에 바로 적용 가능하다. 지금 많은 하네스가 "이 태스크는 deep reasoning이 필요한가?"를 외부에서 분기한다. Phi-4는 그 판단을 모델 내부로 넣었다. 어떤 것이 더 신뢰할 수 있는가?

양쪽에 장단점이 있다:
- 외부 분기: 명시적, 감사 가능, 하네스 제어
- 내부 모드 선택: 암묵적, 더 빠름, 모델에 의존

**2. 오픈소스 멀티모달 추론의 실용화**

이전까지 멀티모달 추론 능력은 GPT-4V, Claude Opus 같은 대형 클라우드 모델에만 있었다. 15B 오픈소스가 이 수준에 근접했다는 것은 로컬 배포 가능한 멀티모달 에이전트가 실용 영역에 진입했음을 의미한다.

**3. 효율적 멀티모달 학습의 레시피 공개**

Microsoft가 technical report로 상세한 ablation 결과를 공개했다. "이것을 하면 얼마나 좋아진다"가 수치로 나와 있다. 이런 공개 레시피는 커뮤니티가 빠르게 따라잡을 수 있게 한다.

### 아직 모르는 것

- THINK/NOTHINK 자동 선택의 실패 케이스 — 모델이 "간단하다"고 잘못 판단하는 경우의 빈도와 패턴
- 데이터 품질 향상의 상한선 — 완전히 합성된 교육 데이터가 인간 생성 데이터를 언제 완전히 대체하는가
- Mid-fusion의 스케일링 특성 — 100B+ 파라미터에서도 같은 효율성 유지가 가능한가

### 더 알고 싶다면

- Technical Report: [arxiv.org/abs/2603.03975](https://arxiv.org/abs/2603.03975)
- Microsoft Research 블로그 (상세한 ablation 포함)

## 이번 주 개념 도구

### 1. 페르소나 선택 (Persona Selection)

LLM은 명령을 실행하는 시스템이 아니라 캐릭터를 시뮬레이션하는 시스템이다. 사전훈련에서 수천 개의 캐릭터를 배우고, 사후훈련이 그 중 "어시스턴트"를 선택한다. 시스템 프롬프트는 명령이 아니라 어떤 캐릭터를 활성화할지를 결정하는 신호다.

*실용적 테스트:* "이 시스템 프롬프트가 어떤 캐릭터를 불러내는가?"라는 질문으로 프롬프트를 평가해보라.

### 2. 적응적 추론 모드 (Adaptive Reasoning Mode)

모든 태스크에 chain-of-thought를 강제하지 않는다. "언제 생각할지"를 명시적으로 제어하는 것 — 모델에게 위임하든, 외부 분기로 구현하든 — 이 추론 비용 최적화의 핵심이다. THINK/NOTHINK는 이것의 모델 내부 구현이다.

*적용:* 에이전트 파이프라인에서 "이 단계는 빠른 분류인가, 깊은 추론인가?"를 명시적으로 분리하라.

### 3. 잠재적 가치 인코딩 (Latent Value Encoding)

Cambridge 연구(arXiv:2603.03000)의 제안: RLAIF(AI 피드백 강화학습)가 작동하는 이유는 사전훈련 데이터에 인간 가치가 이미 표현 공간의 **방향(direction)**으로 인코딩되어 있기 때문이다. Constitutional prompts는 새로운 가치를 주입하는 것이 아니라, 이미 있는 가치 방향을 투영(project)하는 연산자다.

*함의:* RLHF/Constitutional AI의 품질 상한선은 사전훈련 데이터가 인간 가치를 얼마나 잘 인코딩했는가에 달려 있다.

## 시그널 간 연결 — "LLM의 내부 구조"를 이해하려는 흐름

이번 주의 시그널들은 표면적으로 분리되어 있지만 하나의 질문을 향한다: **"LLM이 실제로 무엇을 하고 있는가?"**

PSM은 "LLM은 캐릭터 시뮬레이터"라고 말한다.
RLAIF 이론은 "훈련 데이터에 인간 가치가 방향으로 인코딩되어 있다"고 말한다.
Phi-4의 THINK/NOTHINK는 "모델이 자신의 추론 과정을 모니터링한다"고 시사한다.

세 주장을 함께 놓으면: LLM의 내부 표현은 우리가 생각했던 것보다 훨씬 더 **구조적**이다. 캐릭터, 가치, 추론 전략이 무작위 가중치가 아니라 의미있는 방향으로 조직화되어 있다.

이것이 지금 해석 가능성(interpretability) 연구가 중요한 이유다. 우리가 만드는 하네스는 이 내부 구조 위에서 작동한다. 그 구조를 이해할수록 하네스를 더 잘 설계할 수 있다.

**하네스 엔지니어링과의 연결:**

지난 두 주 동안 우리는 하네스를 "외부에서 모델을 조율하는 시스템"으로 분석했다. PSM은 이것에 새로운 층을 추가한다: 하네스의 핵심 메커니즘 중 하나 — 시스템 프롬프트를 통한 역할 부여 — 가 모델 내부의 페르소나 선택을 통해 작동한다. 외부 조율과 내부 표현의 경계가 생각보다 더 복잡하다.

## 열린 질문들

1. **PSM과 정렬(alignment)**: 만약 Constitutional AI가 "올바른 캐릭터를 선택하도록 사후훈련하는 것"이라면, 정렬의 목표는 무엇인가? 어시스턴트 캐릭터가 좋은 가치를 갖게 하는 것인가, 아니면 모든 잠재적 캐릭터가 좋은 가치를 갖게 하는 것인가?

2. **THINK/NOTHINK의 보정**: 모델이 "이것은 간단하다"고 잘못 판단하면 NOTHINK 모드를 선택한다. 이 오류를 어떻게 탐지하고 교정하는가? 하네스에서 외부 검증이 필요한 케이스를 어떻게 식별하는가?

3. **데이터 품질의 합성 한계**: 데이터를 정제하는 대신 처음부터 합성 데이터를 생성하면 어떻게 되는가? Phi-4의 데이터 레시피를 특정 도메인에 적용하면 — 예: 에이전트 시스템 학습 데이터 — 어느 수준까지 갈 수 있는가?

4. **페르소나 경쟁의 예측**: 멀티에이전트 시스템에서 여러 에이전트가 같은 LLM 기반이면, 그들의 페르소나 선택이 서로 어떻게 간섭하는가? 같은 기반 모델에서 나온 오케스트레이터와 서브에이전트가 묘하게 비슷하게 행동하는 이유가 공유된 캐릭터 레퍼토리 때문인가?

5. **잠재적 가치의 붕괴**: RLAIF 품질 상한선이 사전훈련 데이터의 가치 인코딩 품질에 달려 있다면, 합성 데이터로 훈련된 모델의 RLAIF는 어떻게 작동하는가? 합성 데이터가 가치를 어떻게 인코딩하는가?

## 이번 주 학습 포인트

1. **시스템 프롬프트의 새 멘탈 모델**: 명령이 아니라 캐릭터 활성화 신호. "이 프롬프트가 어떤 존재를 불러내는가?"라는 질문이 "이 프롬프트가 무엇을 시키는가?"보다 더 정확한 질문일 수 있다.

2. **THINK/NOTHINK 패턴 지금 적용**: 에이전트 파이프라인에서 "이 태스크가 deep reasoning이 필요한가?"를 명시적으로 분기하는 것과 모델에게 위임하는 것의 트레이드오프를 의식적으로 설계하라. Phi-4는 후자의 모델 내부 구현이지만, 외부 하네스 수준에서도 같은 원리가 적용된다.

3. **데이터 품질 = 가장 과소평가된 레버**: 240 B200 GPU × 4일로 15B 모델이 이 수준에 도달한 것은 데이터 정제의 힘이다. 파인튜닝이나 특수 목적 에이전트를 만들 때, 데이터를 늘리기 전에 품질을 높이는 것이 우선이다.

4. **해석 가능성 연구가 하네스 설계와 연결된다**: PSM은 순수 연구처럼 보이지만 실용적 함의가 있다. LLM 내부 구조를 더 잘 이해할수록 시스템 프롬프트 설계, 파인튜닝 전략, 에이전트 아이덴티티 관리가 더 원칙적으로 이루어질 수 있다.

5. **RLAIF는 마법이 아니다**: Constitutional AI가 작동하는 이유를 이론으로 설명할 수 있게 됐다. 이것은 RLAIF의 **실패 조건**을 예측할 수 있게 해준다: 사전훈련 데이터가 특정 가치를 제대로 인코딩하지 않은 경우, Constitutional AI로 그 가치를 부여하는 것은 한계가 있다.
